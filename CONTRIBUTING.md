# Contributing

Thanks for your interest in FCuba.

## Reporting issues

Open a GitHub issue using the bug-report or feature-request template.
For security issues please email the maintainer privately rather than
opening a public issue.

## Pull requests

- Open an issue first for non-trivial changes so we can agree on the
  approach before you write code.
- Branch off `main`, keep PRs focused on one thing.
- Match the existing commit-message style (imperative mood, ≤72 chars
  for the subject line, body if needed).
- Keep the diff small. Refactors that are unrelated to the change at
  hand should be a separate PR.

## Local build

See the Developer section of the [README](./README.md) for the full
setup. TL;DR: Node 20+, JDK 21, Android SDK with platform 35; then
`npm ci && npx cap sync android && cd android && ./gradlew assembleDebug`.

CI builds a signed release on every push to `main`, so for most changes
you don't need a local Android toolchain.

## Code style

- Web layer (`www/`): vanilla JS, no transpiler, no framework. Keep
  modules small and avoid global state.
- Native layer (`android/`): generated by Capacitor. Avoid
  hand-edits unless they're guarded by `cap sync` not overwriting them.
